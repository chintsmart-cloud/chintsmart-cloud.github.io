（请求）¶

设备配网（AP）¶

请求URL：

  • http://192.168.0.1

请求方式：

  • POST

请求示例

{
    "message_id": "1bd5d003-31b9-476f-ad03-71d471922820",
    "user_id": "53102fb4bf1044ed8b0ba36a",
    "params": {
        "secret": "53102fb4bf1044ed8b0ba36a",
        "network" : {
            "ssid": "xxxx",
            "pw": "xxxx"
        }
    }
}

请求参数说明

参数名

必选

类型

说明

secret

是

string

设备密钥或产品密钥

network

是

object

用户网络信息

.ssid

是

string

网络id

.pw

是

string

网络密码

返回示例

{
    "error_code": 200,
    "message_id": "1bd5d003-31b9-476f-ad03-71d471922820",
    "data": {
        "device_id": "xxxxx",
        "mac_addr": "xxxxx",
        "sdk_version": "1.0.0",
        "version": "1.0.0"
    }
}

返回数据说明

参数名

类型

说明

device_id

string

设备的唯一id

mac_addr

string

设备物理地址

sdk_version

string

SDK版本

version

string

设备固件版本

注解

message_id 和 error_code 参见 公共参数说明-参数列表

小技巧

本接口支持手动配网和一键配网

设备解绑¶

请求TOPIC：

  • /sys/${product_id}/${device_id}/app/up/thing/unbind

请求方式：

  • MQTT

请求示例

{
    "message_id": "1bd5d003-31b9-476f-ad03-71d471922820",
    "user_id": "53102fb4bf1044ed8b0ba36c",
    "params": {
        "device_id": "53102fb4bf1044ed8b0ba36c"
    }
}

请求参数说明

参数名

必选

类型

说明

device_id

是

string

设备的唯一id

返回数据说明

注解

解绑后，设备将变为 未激活 状态，参见 设备连接状态变更推送， 通过 message_id 标识

更新设备属性¶

请求TOPIC：

  • /sys/${product_id}/${device_id}/app/up/thing/properties

请求方式：

  • MQTT

请求示例

{
    "message_id": "1bd5d003-31b9-476f-ad03-71d471922820",
    "user_id": "53102fb4bf1044ed8b0ba36c",
    "params": {
        "device_id": "53102fb4bf1044ed8b0ba36c",
        "properties": {}
    }
}

请求参数说明

参数名

必选

类型

说明

device_id

是

string

设备的唯一id

properties

是

object

物模型, 和品类id配对, 参考 公共参数说明-物模型

返回数据说明

注解

更新后，参见 设备连接状态变更推送， 通过 message_id 标识
